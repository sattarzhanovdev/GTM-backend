Политика конфиденциальности DBN

Эта политика описывает, какие данные обрабатывает приложение DBN, для чего они используются и как можно запросить удаление аккаунта. Мы не продаем персональные данные третьим лицам и не используем их для показа сторонней рекламы.

1. О приложении

Приложение DBN предназначено для жителей жилого комплекса. Через него можно получать уведомления, просматривать начисления и оплаты, а также использовать функции доступа, если они доступны для конкретного комплекса.

2. Какие данные мы обрабатываем

  • Данные для входа, включая номер квартиры и номер подъезда, если они используются для авторизации и привязки аккаунта.
  • Токен push-уведомлений устройства, чтобы доставлять уведомления через Firebase Cloud Messaging (FCM).
  • Файлы чеков и связанные сведения, если пользователь загружает чек для проверки оплаты.
  • Технические данные сервера, включая дату, время запросов и служебные логи, необходимые для диагностики и защиты сервиса.

3. Разрешения Android

Приложение может запрашивать разрешения устройства только для работы нужных функций:

  • POST_NOTIFICATIONS для отправки уведомлений.
  • READ_EXTERNAL_STORAGE и WRITE_EXTERNAL_STORAGE для выбора и загрузки файлов, например чеков, если эта функция используется на устройстве.
  • RECORD_AUDIO только если в приложении используется сценарий, связанный с записью аудио или системными возможностями, которым нужен доступ к микрофону.

4. Зачем мы используем данные

  • Чтобы авторизовать пользователя и показывать данные его квартиры.
  • Чтобы отправлять сервисные и информационные уведомления.
  • Чтобы принимать и проверять загруженные чеки.
  • Чтобы поддерживать безопасность, стабильность и работоспособность сервиса.

5. Кому могут передаваться данные

  • Google Firebase Cloud Messaging, если это необходимо для доставки push-уведомлений.
  • Администратору жилого комплекса или уполномоченным сотрудникам, которые обрабатывают обращения, платежи и загруженные пользователем документы.

6. Срок хранения

  • Токен push-уведомлений хранится, пока пользователь использует приложение и уведомления активны.
  • Чеки и связанные данные хранятся на период проверки и дальнейшего обслуживания по внутренним правилам комплекса.
  • Данные аккаунта хранятся, пока аккаунт остается активным.

7. Удаление аккаунта и данных

В приложении доступна функция удаления аккаунта, которая отправляет запрос администратору. После обработки запроса аккаунт и связанные с ним данные удаляются или обезличиваются в течение 1 дня, если иное не требуется по закону или для завершения расчетов и проверки платежей.

8. Безопасность

Мы принимаем разумные организационные и технические меры для защиты данных от несанкционированного доступа, изменения, раскрытия или уничтожения.
